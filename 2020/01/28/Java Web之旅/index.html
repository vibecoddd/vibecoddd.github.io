MVC框架选型

在开始Java Web之旅前，我们先聊聊MVC框架的选型。

主流MVC框架对比

传统三大框架：

Spring MVC

  • 优点：Spring生态核心组件，社区活跃，文档完善，企业级应用首选
  • 缺点：配置繁琐（XML地狱），需要手动配置大量组件
  • 适用场景：大型企业级应用，需要精细控制的项目

Struts 2

  • 优点：曾经的王者，插件丰富，拦截器机制强大
  • 缺点：性能一般，安全漏洞频发，社区活跃度下降
  • 适用场景：老项目维护（新项目不推荐）

JSF (JavaServer Faces)

  • 优点：Java EE标准，组件化开发
  • 缺点：学习曲线陡峭，灵活性不足
  • 适用场景：重度依赖Java EE的项目

新生代框架：

Spring Boot

  • 优点：约定优于配置，开箱即用，快速开发，微服务友好
  • 缺点：封装较深，初学者可能不了解底层原理
  • 适用场景：现代化Web应用、微服务架构、快速原型开发

Play Framework

  • 优点：异步非阻塞，性能优秀，支持Scala
  • 缺点：生态不如Spring，企业采用率较低
  • 适用场景：高并发Web应用

Vert.x

  • 优点：响应式编程，轻量级，性能极佳
  • 缺点：学习曲线陡峭，生态较小
  • 适用场景：高性能、高并发场景

为什么选择Spring Boot？

根据2020年的技术趋势，Spring Boot已成为Java Web开发的事实标准，原因如下：

1. 约定优于配置
告别XML配置地狱，Spring Boot通过智能默认配置和自动装配机制，让你专注于业务逻辑而非繁琐的配置。一个注解就能搞定传统Spring MVC需要十几行XML的配置！

2. 开箱即用
内嵌Tomcat/Jetty/Undertow服务器，无需部署WAR包，直接java -jar运行。这种”胖jar”模式特别适合微服务和容器化部署。

3. 微服务友好
Spring Boot + Spring Cloud是构建微服务架构的黄金组合。服务发现、配置中心、熔断限流、API网关等微服务组件一应俱全。

4. 生产就绪
内置健康检查、指标监控、外部化配置等生产级特性，通过Spring Boot Actuator可以轻松监控应用状态。

5. 生态强大
Spring生态覆盖数据访问、安全、批处理、消息队列等各个领域。遇到问题？Stack Overflow上有海量答案。

6. 快速开发
Spring Initializr一键生成项目骨架，加上热部署工具devtools，开发效率翻倍！

Spring Boot核心特性

自动配置（Auto-Configuration）
Spring Boot会根据classpath中的依赖自动配置应用。比如，加入spring-boot-starter-web依赖，它会自动配置DispatcherServlet、视图解析器、JSON转换器等组件。

起步依赖（Starter Dependencies）
预定义的依赖集合，避免版本冲突。比如spring-boot-starter-web会自动引入Spring MVC、Tomcat、Jackson等所有Web开发需要的依赖。

内嵌服务器
不再需要外部Tomcat，应用自带服务器，部署时只需要一个jar包。这极大简化了部署流程，特别适合云原生应用。

Actuator监控
生产级的监控端点，可以查看健康状况、配置信息、日志、JVM指标等。结合Prometheus + Grafana可以构建完整的监控体系。

Spring Boot vs 传统Spring MVC

特性 Spring MVC Spring Boot
配置方式 XML或Java Config 自动配置 + 注解
项目搭建 手动配置，耗时 Spring Initializr，秒级
依赖管理 手动管理版本 Starter统一版本
服务器部署 需要外部Tomcat 内嵌服务器
开发效率 较低 高
学习曲线 陡峭 平缓
适用场景 传统企业应用 现代化应用、微服务

简单来说：Spring Boot = Spring MVC + 自动配置 + 内嵌服务器 + 开箱即用

使用插件新建Maven项目

选择Spring Assistant按步骤新建项目，这个过程和使用Spring Initializr官网基本一致。

Step 1: 选择项目类型和Spring Boot版本

  • Type: 选择Maven Project（也可以选Gradle，看个人喜好）
  • Language: Java
  • Packaging: Jar（推荐，方便部署）
  • Java Version: 8或11（根据你安装的JDK版本选择）
  • Spring Boot Version: 选择稳定版本，避免SNAPSHOT版本

Step 2: 配置项目元数据

  • Group: 公司域名反写，如com.example
  • Artifact: 项目名称，如demo01
  • Package Name: 自动生成，也可以手动修改
  • Name: 项目显示名称
  • Description: 项目描述（可选）

这些信息会影响Maven的坐标和Java包结构，建议认真填写。

Step 3: 选择项目依赖

Spring Boot提供了上百个Starter依赖，初学者可以先选择：

  • Spring Web: 包含Spring MVC和Tomcat，Web开发必选
  • Spring Boot DevTools: 热部署工具，修改代码自动重启（可选但推荐）
  • Lombok: 简化实体类代码（可选）

随着学习深入，后续可以添加：

  • Spring Data JPA（数据库ORM）
  • MySQL Driver（数据库驱动）
  • Spring Security（安全框架）
  • Thymeleaf（模板引擎）

Step 4: 确认项目信息并创建

检查项目路径和名称，点击Finish，IDEA会自动下载Maven依赖。第一次下载可能需要几分钟，耐心等待！

Tips： 如果Maven下载依赖很慢，可以配置国内镜像源（阿里云、华为云等），在Maven的settings.xml中添加mirror配置。

@RestController注解说明：

  • @RestController = @Controller + @ResponseBody
  • @Controller: 标识这是一个Spring MVC控制器
  • @ResponseBody: 将返回值直接写入HTTP响应体，而非渲染视图
  • 适用于RESTful API开发，返回JSON/XML数据

如果你要返回HTML页面（使用Thymeleaf等模板引擎），应该使用@Controller而非@RestController。

添加请求映射方法

使用**@RequestMapping**注解设置URL映射规则，并创建handler方法：

@RequestMapping详解：

  • value或直接写路径：指定URL路径
  • method: 指定HTTP方法（GET、POST、PUT、DELETE等）
  • params: 指定请求参数
  • headers: 指定请求头

更推荐的写法（RESTful风格）：

注解对比：

  • @GetMapping = @RequestMapping(method = RequestMethod.GET)
  • @PostMapping = @RequestMapping(method = RequestMethod.POST)
  • @PutMapping = @RequestMapping(method = RequestMethod.PUT)
  • @DeleteMapping = @RequestMapping(method = RequestMethod.DELETE)
